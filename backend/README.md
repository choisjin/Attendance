# 팀 근태 관리 서버

노트북 한 대를 팀 서버로 띄우면, 팀원들이 같은 네트워크에서 **웹 주소(노트북 IP)** 로 접속해
함께 사용하는 도구입니다. 이 서버 한 곳이:

1. **웹 페이지 서빙** — `attendance-app.html`을 `http://<노트북 IP>:포트/` 로 제공
2. **데이터 공유 저장** — 모든 사용자의 수정·싱크·저장을 서버 파일(`data.json`)에 저장.
   누가 어디서 바꿔도 모두에게 반영되고, **서버를 재시작해도 그대로 유지**됩니다.
3. **그룹웨어 근태 싱크** — 헤드리스 브라우저(Puppeteer)로 그룹웨어에 대신 로그인해 근태를 가져옵니다.

## 팀 서버로 실행하기 (노트북에서)

```bash
cd backend
npm install     # 최초 1회 (Puppeteer가 Chromium 다운로드 — 수십 MB)
npm start
```

실행하면 콘솔에 접속 주소가 표시됩니다:

```
 · 이 노트북:   http://localhost:3939/
 · 팀원 접속용: http://192.168.x.x:3939/      ← 이 주소를 팀원에게 공유
```

- 팀원은 같은 사내망/Wi‑Fi에서 위 **팀원 접속용 주소**를 브라우저에 입력해 사용합니다.
- **Windows 방화벽**: 최초 실행 시 "액세스 허용" 창이 뜨면 허용하세요. 안 떴는데 팀원이 접속 안 되면,
  방화벽에서 이 포트(기본 3939) **인바운드 허용**이 필요합니다.
- 포트 변경: `PORT=80 npm start` (80이면 주소에 포트 생략 가능, 관리자 권한 필요할 수 있음)
- 데이터 파일 위치 변경: `DATA_FILE=D:/att/data.json npm start`
- 노트북은 켜져 있어야 팀원이 접속할 수 있습니다(서버 역할).

## 데이터 저장 / 동기화

- 앱의 모든 상태(인원·프로젝트·근태·보고 사유 등)는 `backend/data.json` 한 파일에 저장됩니다.
  **백업하려면 이 파일을 복사**하면 됩니다.
- 각 브라우저는 5초마다 서버를 확인해 다른 사람이 바꾼 내용을 자동 반영합니다.
- **로그인 상태**와 **그룹웨어 ID/PW**는 보안상 서버가 아니라 각자 브라우저(기기)에만 저장됩니다.
  (그룹웨어 비밀번호는 싱크 시에만 서버로 전송돼 메모리에서 쓰인 뒤 폐기, 저장 안 함)

## 그룹웨어 싱크 동작 방식

1. 프론트가 저장된 **그룹웨어 ID/PW**와 조회할 월을 이 서버로 POST
2. 서버가 Puppeteer(헤드리스 Chrome)로 `https://gw.linkgenesis.co.kr` 로그인
   - 아이디 `#log-userid`, 비밀번호 보안 iframe `#iframeLoginPassword > #p`, 로그인 `#btn-log`
3. 로그인 세션으로 근태 JSON API 호출
   - `GET /nhr/api/timecard/user/schedule/calendar?day=YYYY-MM` (월별 출퇴근/이벤트)
   - 퇴근이 17:30 이후인 날만 `GET /nhr/api/timecard/user/status/day?day=YYYY-MM-DD` (연장근무 확인)
4. **지각**: `출근/지각` 기록과 `schedule`(반차 반영 기준시각)으로 분 계산
   **조퇴**: `조퇴 HH:MM` 기록이 있는 날의 조기퇴근 분(기준 퇴근 − 실제 퇴근) 계산
   (앱에서 조퇴는 적립된 추가근무로 상계 등록 가능)
5. **추가근무(연장근무)**: 그룹웨어가 인정한 값으로만 등록
   - `status/day`의 `analysis.over_time_label`(연장근무) / `night_work_label`(야간근무)이 있을 때만
   - 그리고 **실제 퇴근 펀치일 때만** (`punch.out.time == punch.out.real_time`).
     관리자가 보정해 넣은 가짜 퇴근(예: 미클럭아웃 보정)으로 생긴 소액 연장근무는 제외.
   - 즉 "퇴근시각이 17:30 이후"라는 이유만으로는 추가근무로 인정하지 않음.

표준 근무 기준(사용자 확정): **08:30~17:30**, **오전 반차 13:30**, **오후 반차 12:30**.

## API

### 데이터 (공유 저장)
- `GET /api/data` → `{ ok, data, rev }` 현재 공유 데이터
- `PUT /api/data` (body: `{ data }`) → 저장 후 `{ ok, rev }`. 서버가 `rev`(리비전)를 부여하며,
  각 브라우저는 이 값으로 변경을 감지해 자동 새로고침합니다.

### 페이지
- `GET /` 또는 `GET /attendance-app.html` → 웹 앱 (그 외 파일은 서빙하지 않음)

### `POST /api/groupware/attendance`

요청
```json
{ "id": "그룹웨어ID", "pw": "그룹웨어PW", "months": ["2026-04", "2026-06"] }
```
(`month` 단일 문자열도 허용)

응답
```json
{
  "ok": true,
  "name": "",
  "months": {
    "2026-06": [
      {
        "date": "2026-06-10", "weekday": "수", "isHoliday": false,
        "event": "", "halfDay": "",
        "schedStart": "08:30", "schedEnd": "17:30",
        "checkIn": "13:22", "checkOut": "17:30",
        "gwLate": true, "gwEarly": false,
        "lateMinutes": 292, "overtimeMinutes": 0
      }
    ]
  }
}
```

로그인 실패 시 `401`, 잘못된 요청 시 `400` + `{ ok:false, code, error }`.

## 보안 주의

- **사내망 내부 도구**입니다. 인터넷에 직접 공개(포트포워딩 등)하지 마세요.
- 통신은 평문 HTTP입니다. 그룹웨어 ID/PW는 싱크 시에만 전송돼 서버 메모리에서 쓰인 뒤 폐기(미저장).
- `data.json`에는 인원 정보·근태가 들어가니 노트북/파일 접근을 관리하세요. (그룹웨어 비밀번호는 미포함)

## 참고 / 한계

- 동시 편집은 "마지막 저장 우선(last-write-wins)"입니다. 같은 항목을 여러 명이 몇 초 안에
  동시에 바꾸면 한쪽이 덮어쓸 수 있습니다(소규모 팀 사용에는 충분).
- 싱크는 **서버 노트북**에서 헤드리스 브라우저로 실행되므로, 팀원 PC에는 Chromium이 필요 없습니다.
